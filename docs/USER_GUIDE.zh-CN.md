# 时间盒使用手册

本文档面向使用开源版 Timeboxing 的用户，说明每个页面的功能、使用方式和常见操作路径。

## 核心概念

- 时间盒：一段有开始时间、结束时间和明确任务标题的时间块。
- 待办事项：还没有安排到具体时间的任务，位于计划页左侧。
- 计划盒：每天开始时用于制定今日计划的时间盒，可在计划页提示中创建。
- 空窗：工作日范围内没有被其他时间盒占用的时间段。
- 状态：`已计划`、`进行中`、`已完成`、`未完成`。
- 紧急程度：通过标签设置，支持 `紧急`、`重要`、`一般`，并影响日历卡片颜色。
- 本地数据：所有数据默认保存在当前设备的 IndexedDB 中，不需要服务器。

## 推荐工作流

1. 在 `Settings 设置` 里确认工作日开始、结束时间。
2. 进入 `Plan 计划`，添加待办事项和预估分钟数。
3. 选中待办，设置紧急程度，再点击 `安排到今天`。
4. 在日历中检查时间盒位置，拖拽或拉伸调整时间。
5. 进入 `Focus 执行`，按当前时间开始执行。
6. 工作结束后进入 `Review 复盘`，处理未完成项并查看完成情况。
7. 定期在 `Settings 设置 -> 本地数据` 导出 JSON 备份。

## 全局布局

### 顶部日期栏

- `日期` 输入框用于切换当前查看的日期。
- 日期范围限制为 `2000-01-01` 到 `2099-12-31`。
- `今天` 按钮会快速回到当天。
- 当前日期会影响 `Plan 计划`、`Focus 执行`、`Review 复盘` 和 `Settings 设置` 中显示的数据。

### 左侧导航

- `Plan 计划`：创建待办、安排时间盒、调整日历。
- `Focus 执行`：启动和推进当前时间盒。
- `Review 复盘`：查看当天完成、未完成和计划中的时间盒。
- `Settings 设置`：配置工作时间、专注保护和本地数据。

## 首页

路径：`/`

首页提供四个快速入口：

- `开始计划`：进入 `Plan 计划`。
- `专注模式`：进入 `Focus 执行`。
- `回顾总结`：进入 `Review 复盘`。
- `设置`：进入 `Settings 设置`。

首页不保存数据，主要用于快速跳转。

## Plan 计划

路径：`/plan`

计划页是主要工作区，由左侧待办事项、中间日历、右侧属性面板组成。

### 创建今日计划盒

如果当天还没有计划盒，页面顶部会显示提示：

- `创建计划盒`：在工作日开始时间创建一个“制定今日计划”的时间盒。
- `今日忽略`：只关闭提示，不创建计划盒。

计划盒默认时长来自 `Settings 设置 -> 每日计划时长（分钟）`。

### 待办事项列表

左侧区域用于管理还没有安排到具体时间的任务。

可用操作：

- 输入 `待办标题` 和预估分钟数，点击 `添加`。
- 预估分钟数范围是 `5` 到 `480` 分钟。
- 勾选多个待办后，可点击 `重要`、`紧急`、`一般` 批量设置紧急程度。
- 勾选多个待办后，点击 `安排到今天`，系统会按工作日起点和空窗自动安排。
- 勾选多个待办后，点击 `批量删除` 可删除这些待办。
- 点击某个待办行，会显示 `编辑` 和 `删除`。
- 编辑时可修改标题和预估分钟数。

待办列表按中文拼音和数字顺序展示。安排到今天时，系统会优先安排更紧急的任务。

### 待办快捷键

当某一行待办已经展开操作按钮时：

- `E`：进入编辑。
- `Delete` 或 `Backspace`：删除该待办。
- `Esc`：收起操作按钮。

当正在编辑待办时：

- `Enter`：保存。
- `Esc`：取消编辑。

### 中间日历

中间日历显示当前日期的时间盒，分为两栏：

- 左栏：`05:00-13:00`
- 右栏：`13:00-24:00`

时间盒颜色含义：

- 红色系：紧急或未完成。
- 黄色系：重要或默认计划。
- 绿色系：一般或已完成。
- 蓝色系：计划盒。

可用操作：

- 点击时间盒：选中该时间盒，并在右侧显示属性。
- 长按时间盒左侧区域约 300ms 后拖拽：移动时间盒。
- 拖拽时间盒底部：调整结束时间和时长。
- 点击空白处：取消选中。
- 如果移动或拉伸造成时间冲突，会出现冲突提示。

冲突提示提供三个选择：

- `移到下一空窗`：自动移动到下一个可用时间段。
- `仍然保存`：保留冲突时间。
- `取消`：放弃本次调整。

### 日历卡片操作

选中时间盒后，卡片附近会显示操作菜单。不同状态可用操作不同：

- `已计划`：开始、顺延、删除。
- `进行中`：完成、分割、删除。
- `未完成`：顺延、删除。
- `已完成`：删除。

### 日历快捷键

选中一个时间盒后：

- `Space` 或 `Enter`：已计划时开始，进行中时完成。
- `X`：分割进行中的时间盒。
- `S`：顺延已计划的时间盒。
- `Esc`：取消选中。

### 属性面板

右侧属性面板用于编辑当前选中的时间盒。

可查看：

- 当前时长。
- 当前状态。

可编辑：

- 标题。
- 标签，多个标签用空格分隔。
- 备注。
- 紧急程度：`重要`、`紧急`、`一般`。

修改标题、标签或备注后，点击 `保存属性`。如果没有改动，保存按钮会保持禁用。

状态操作：

- `开始`：把已计划时间盒改为进行中。
- `完成`：把时间盒改为已完成。
- `删除`：删除该时间盒。
- `顺延`：移动到下一空窗。
- `分割`：将进行中的时间盒切成“已完成部分”和“剩余待做部分”。

## Focus 执行

路径：`/focus`

执行页用于在工作时推进当前时间盒。

页面会根据当前时间和当前日期显示三种情况：

- 有 `进行中` 时间盒：显示当前任务、时间范围和剩余分钟数。
- 当前时间命中一个 `已计划` 时间盒：显示“当前时段应做”的任务。
- 当前没有命中任务但还有未来任务：显示“下一个”任务。

### 无进行中时间盒时

可用操作：

- `开始`：开始当前应做或下一个已计划时间盒。
- `顺延`：把该时间盒移动到下一空窗。

### 有进行中时间盒时

可用操作：

- `完成`：标记为已完成。
- `分割`：把当前已执行部分标记完成，并把剩余时长安排到下一空窗。

### 执行页快捷键

- `Space` 或 `Enter`：有进行中时间盒时完成；没有进行中但有应做任务时开始；否则开始下一个计划任务。
- `X`：分割进行中的时间盒。
- `S`：顺延当前应做或下一个计划任务。
- `5`：将进行中时间盒延长 5 分钟。
- `0`：将进行中时间盒延长 10 分钟。

如果延长造成时间冲突，页面会显示冲突提示，可选择：

- `仍然延长`
- `移到下一空窗`
- `取消`

### 专注保护

当 `Settings 设置 -> 专注保护` 开启，并且存在进行中时间盒时：

- 页面会显示专注保护提示。
- 部分提示和快捷键会被屏蔽。
- `完成` 仍可使用，确保可以结束当前任务。

## Review 复盘

路径：`/review`

复盘页用于查看当天执行结果。进入页面或点击 `刷新` 时，系统会把已经超过结束时间且仍为已计划的时间盒标记为 `未完成`。

### 指标区

顶部显示四个指标：

- `已计划盒子`：仍处于已计划状态的数量和分钟数。
- `已完成盒子`：已完成数量和分钟数。
- `效率（完成/计划）`：已完成分钟数占当天全部计划分钟数的比例。
- `进行中`：当前是否有正在推进的时间盒。

### 今日已完成

显示当天所有已完成时间盒，包括标题、时间范围和分钟数。

### 今日未完成（超时）

显示当天已经超时但未完成的时间盒。

可用操作：

- `一键顺延全部`：把所有未完成项依次顺延到后续空窗。
- 单项 `顺延`：只顺延该未完成时间盒。
- 单项 `删除`：删除该未完成时间盒，删除前会二次确认。

### 今日已计划

显示当天仍未开始或未完成的计划时间盒。

## Settings 设置

路径：`/settings`

设置页包含工作参数和本地数据管理。

### 工作日与计划参数

可配置：

- `工作日开始`：自动安排时间盒时的起点。
- `工作日结束`：自动安排时间盒时的终点。
- `每日计划时长（分钟）`：创建计划盒时使用的默认时长，范围是 `5` 到 `240` 分钟。
- `专注保护`：执行页有进行中时间盒时减少提示干扰。

按钮：

- `保存设置`：保存当前设置。
- `恢复默认`：恢复应用默认设置。

### 本地数据

这里显示当前本地数据统计：

- 数据库名称。
- 数据库版本。
- 时间盒数量。
- 待办数量。
- 日志数量。

可用操作：

- `导出 JSON`：下载完整本地数据备份。
- `导入 JSON`：选择备份文件并恢复数据。
- `清空数据`：清空时间盒、待办、日志和设置，并恢复默认设置。

导入会替换当前本地数据。建议导入前先导出一份备份。

## 数据备份建议

- 在大规模调整计划前，先导出 JSON。
- 在不同浏览器或 Electron 桌面版之间迁移时，使用导出和导入。
- 备份文件包含时间盒、待办、日志和设置，不包含任何服务器账号信息。

## 常见问题

### 是否需要登录？

不需要。开源版默认本地使用。

### 数据会上传到服务器吗？

不会。默认数据保存在当前设备的 IndexedDB 中。

### 为什么顺延不一定还在今天？

如果今天没有足够空窗，顺延会继续查找未来最多 7 天的可用时间段。

### 为什么有的计划会变成未完成？

复盘页刷新时，会把已经超过结束时间并且仍为已计划的时间盒标记为未完成。新创建的时间盒有短暂缓冲，不会立刻被标记。
